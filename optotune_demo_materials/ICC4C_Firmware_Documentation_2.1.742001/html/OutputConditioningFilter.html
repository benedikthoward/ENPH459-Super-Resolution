ICC4C  143-525-00
Input/Output Conditioning Filter

Input conditioning filter system base ID: 0x9C
Output conditioning filter system base ID: 0xD8

Input/output conditioning filter is a part of input/output conditioning stage of Signal flow.
List of main features:

  • stage can use up to four filter instances in series.
  • filter instance can be configured as FIR or IIR filter or their combination in series
  • each filter instance can have up to 100 coefficients for input and output sample status
  • loading one filter instance using communication buffers preloaded by vector data communication
  • activation/deactivation of filters is via mask of active filters, in system registers.
  • default filter setting includes coefficients value a0=1.0 and b0=1.0 , to simply propagate input sample to output without modification

Stage system values

Reading system input/output values within signal flow

  • getting stage system input available via registers inputValue [RegId:0x10]
  • getting stage system output available via register outputValue [RegId:0x20]
  • intermediate values of individual filter instances in cascade are available via
    stage system status registers inputStatus [RegId:0x1[1-4]]
    stage system status registers outputStatus [RegId:0x2[1-4]]

Note: Getting status value from non-active filter within stage is possible.
Non active filter value is not contributing to stage output, and obviously is not changing.

Downloading filter coefficients to driver

Filter instances shall be loaded one by one. Activation and deactivation of filter instances has no affect on loaded data.
Downloading actual filter instance coefficients uses simple protocol state machine:

  • set register:
    • selectedFilterComm [RegId:0x01] with value (0, 1, 2, 3) as an index of filter instance
  • set vector: 1), 2), 3)
    • inFilterCoeffComm [VectId:0x00] - loading filter input sample coefficients
  • set vector: 1), 2), 3)
    • outFilterCoeffComm [VectId:0x01] - loading filter output sample coefficients
  • set register:
    • loadFilterCoeff [RegId:0x02] with value (false/true)
      • false - loads filter coefficients only, filter continue with running calculation
      • true - loads filter coefficients and erases filter internal status additionally, i.e. filter output starts from zero

Note: If any error appears during coefficients loading, protocol has to start from beginning.

Uploading filter coefficients from driver

Uploading actual filter coefficients set within firmware is possible by getting vector data within recommended command sequence:

  • set register:
    • selectedFilterComm [RegId:0x01] with value (0, 1, 2, 3) as an index of filter instance
  • get register:
    • inCoeffCommNbr [RegId:0x03] - to get actual number of selected filter input sample coefficients
    • outCoeffCommNbr [RegId:0x04] - to get actual number of selected filter output sample coefficients
  • get vector: 1), 2), 3)
    • inFilterCoeffComm [VectId:0x00] - getting filter input sample coefficients
  • get vector: 1), 2), 3)
    • or outFilterCoeffComm [VectId:0x01] - getting filter output sample coefficients


Remark:
1) coefficients values can be uploaded/downloaded within a sequence of vectors
2) one vector is limited to 11 coefficient values
3) max addressable coefficient number is 99, i.e 100 coefficient items in total, counting from zero

Register Map:

Address Name Default Description Type Access
0x[SysId]00 mskActiveFilters 0.0 mask of active filters uint read/write
0x[SysId]01 selectedFilterComm 0 select filter for coeff. reception uint read/write
0x[SysId]02 loadFilterCoeff 0 cmd: load coeff. to filter bool write only
0x[SysId]03 inCoeffCommNbr 1 read selected filter input coeffs. uint read only
0x[SysId]04 outCoeffCommNbr 1 read selected filter output coeffs. uint read only
0x[SysId]10 inputValue 0.0 OCF input value float read only
0x[SysId]1[1-4] inputStatus[0-3] 0.0 input value of particular filter float read only
0x[SysId]20 outputValue 0.0 OCF output value float read only
0x[SysId]2[1-4] outputStatus[0-3] 0.0 output value of particular filter float read only

Vector Map:

Address Name Default Description Type Access
0x[SysId]00 inFilterCoeffComm 0.0 buffer of input coeffs from comm. float array read/write
0x[SysId]01 outFilterCoeffComm 0.0 buffer of output coeffs from comm. float array read/write

ICC4C  143-525-00
Waveform Adaptation Unit (Iterative Control)

Waveform Adaptation Unit system base ID: 0x33

The Waveform Adaptation Unit implements an Iterative Control algorithm for two axes.

The unit interacts with WindowAverager. Inputs from WindowAverager are the following:

  • sensor data averaged over several repetition cycles (windows). Averaging is done to reduce noise.
  • window length (based on internal or external trigger)

The unit interacts with the Vector Pattern Memory :

  • reading the transition template vector from VectorPatternMemory
  • storing the generated setpoint curves into VectorPatternMemory
  • storing the calculated actuation waveform into the VectorPatternMemory. This signal is the used by the Vector Pattern Unit.

For the unit to work correctly certain memory ranges need to be available in Vector Pattern Memory. An Example memory map is:

Address Range Data Content
0 - WINDOW_LEN Setpoint waveform for channel 0 (SOURCE_ADDRESS = 0)
500 - (500 + WINDOW_LEN) Setpoint waveform for channel 1 (SOURCE_ADDRESS = 0)
1000 - (1000 + WINDOW_LEN) Actuation Waveform for channel 0 (TARGET_ADDRESS = 1000)
1500 - (1500 + WINDOW_LEN) Actuation Waveform for channel 1 (TARGET_ADDRESS = 1000)
2000 - (2000 + WINDOW_LEN) Measured Waveform for channel 0 (source_address_measurement = 2000)
2500 - (2500 + WINDOW_LEN) Measured Waveform for channel 1 (source_address_measurement = 2000)
3000 - 3200 Transition Template (source_address_transition_template = 3000)

Note:

  • The transition template, which is a curve describing a smooth step, needs to be stored before running this unit (e.g. at startup).
  • The transition templates are used to calculate the setpoint waveforms. Setpoint waveforms depend on window length, amplitude, rotation, dark time, phase shift.
  • The measured waveform data is constantly updated by the WindowAverager.
  • The setpoint waveform and the actuation waveform are generated by this unit.

The unit interacts with the Linear Output :

  • to disable the outputs in case of a "safety check" event (measured movement too high, trigger frequency change detected, etc). The outputs are automatically re-enabled once the problem disappears.

Register Map:

Address Name Default Value Description Type Access
0x3300 Run 0 0: Disabled, 1: Run for one iteration, 2: Run continuously uint16 read write
0x3302 SOURCE_ADDRESS 0 Address in the VPM where the setpoints waveforms are stored (for second channels SOURCE_ADDRESS +500) uint16 read write
0x3303 TARGET_ADDRESS 0 Address in the VPM where the actuation waveforms are stored (for second channels TARGET_ADDRESS +500) uint16 read write
0x3304 WINDOW_LEN - Window Length (automatically read from WindowAverager) uint16 read
0x3305 DEBUG 0 0: Normal Mode, 1: Debug Mode boolean read write
0x3306 EPSILON 0 Epsilon Parameter of the algorithm float read write
0x3307 LEARNING_FACTOR 0.3 Learning Factor of the algorithm float read write
0x3308 TT_LEN 200 Transition Template Length uint16 read write
0x3309 AMPLITUDE 0.15 [deg] Amplitude of Setpoint Waveform float read write
0x330a ANGLE 0 [rad] Rotation of the Setpoint waveform (in radiant) float read write
0x330b CONFIG 0 Configuration for debugging (Bit 1: don't generate setpoints) uint16 read write
0x330c NUMBER_OF_ITERATIONS - Number of iterations performed (since last safety restart) uint32 read
0x330d RMS_SPEC 0.045 RMS Error specification (used for OUT_OF_SPEC_ITERATIONS calculations) float read write
0x330e SAFETY_MAX_OUT 0.55 Maximum allowed value for actuation waveform. Waveform is scaled down if calculation yields higher value. float read write
0x330f OUT_OF_SPEC_ITERATIONS 0 Number of iterations with RMS error > RMS_SPEC uint32 read
0x3310 NUM_SAFETY_TRIGGERES 0 Number of safety events which occurred (in that case iteration process restarts) uint32 read write
0x3311 RMS ERROR - [deg] RMS error (1 if not moving) float read
0x3312 PHASE_SHIFT 0 [ms] Phase shift of the setpoint trajectory float read write
0x3313 INITIAL_AMPLITUDE_FACT 0.1 Scaling factor for the waveform of the first iteration float read write
0x3314 F_CUTOFF 650 [Hz] Cutoff frequency in Hz for the actuation waveform. All frequency components above are set to 0. float read write
0x3315 PHI_C 0 [rad] Rotation angle for the coordinate system in which the algorithm operates. Should ideally correspond to the mode angles. float read write
0x3316 DYNAMIC_AMPLITUDE - Maximum value of the actuation waveforms float read
0x3317 RMS_Z - [mm] RMS Error in z float read
0x3318 DARK_TIME 0 [ms] Dark time, for dark time >0 set SECOND_ORDER_CORRECTION to 0 or 3 for faster convergence float read write
0x3319 TRANSTION_TIME 1.1 [ms] Transition time of the setpoint waveform float read write
0x331a RMS_ERROR_MAX - Maximum RMS Error Measured up to now float read
0x331b RESET_STATISTCS - Writing 1 resets: NUMBER_OF_ITERATIONS, OUT_OF_SPEC_ITERATIONS, NUM_SAFETY_TRIGGERES, RMS_ERROR_MAX boolean write
0x331c SAFETY_ERROR_CODE - Reason for last safety trigger event (see below) uint16 read
0x331d SECOND_ORDER_CORRECTION 0 Control the second harmonics (0: deactivated, 1: activated, 3: hybrid (only activated if RMS ERROR>RMS_SPEC) boolean read write
0x331e INITIAL_SECOND_HARM 0.03 Magnitude of second harmonics signal for first second harmonics iteration float read write
0x331f RESET_ITERATION - Write to this register to restart the iteration bool write
0x3320 ALLOWED_MOVEMENT_RANGE 1.6 Write to this register to restart the iteration bool write
0x3321 MODE 0 Sets 4P (=0) or 2P (=1) mode bool write
  • The SAFETY_ERROR_CODE description is:
    Value Description
    0 No safety trigger
    1 OVER_TEMPERATURE
    2 OUTPUT_CURRENT_TOO_HIGH
    3 SENSOR_AMPLITUDE_TOO_HIGH
    4 FREQUENCY_OUT_OF_RANGE
    5 HF_READOUT_ERROR
